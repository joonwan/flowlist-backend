# Flow List

Flow List는 만들면서 배우기 위한 AI 기반 태스크 관리 백엔드 프로젝트입니다.

단순한 예제를 따로따로 따라가는 대신, 이 프로젝트는 하나의 FastAPI 애플리케이션을 점진적으로 확장해 나가는 방식으로 구성됩니다. 처음에는 간단한 TODO API로 시작하지만, 점차 영속성, 비동기 워크플로우, 캐싱, 그리고 AI 기반 기능까지 포함하는 더 똑똑한 시스템으로 발전시켜 나갈 예정입니다.

## 이 프로젝트를 만드는 이유

이 저장소는 FastAPI를 손으로 직접 구현하며 익히기 위한 실습 프로젝트입니다.

목표는 실제 백엔드를 단계적으로 만들어 가면서 자연스럽게 아래 주제들을 익히는 것입니다.

- FastAPI 기반 REST API 설계
- 데이터 모델링과 데이터베이스 연동
- 백그라운드 및 이벤트 기반 워크플로우
- 캐시를 통한 성능 개선
- AI 기반 태스크 기능 설계

각 주제를 따로 떼어 학습하는 대신, Flow List는 이것들을 하나의 제품 아이디어 안에 모읍니다. 복잡하고 흩어진 계획을 더 명확하고 실행 가능한 흐름으로 바꿔 주는 태스크 매니저가 그 출발점입니다.

## 프로젝트 비전

Flow List는 단순한 태스크 관리 API로 시작하지만, 점차 다음과 같은 기능을 갖춘 지능형 백엔드로 확장됩니다.

- 태스크 생성, 수정, 완료, 정리
- 우선순위, 마감일, 상태 기준 분류
- 자주 조회되는 화면을 빠르게 제공하는 조회 구조
- 태스크 이벤트 발생 시 비동기 워크플로우 실행
- 태스크를 다듬고, 쪼개고, 요약하는 AI 기능 지원

## 예상 학습 흐름

프로젝트는 대략 아래와 같은 단계로 성장할 예정입니다.

1. FastAPI로 기본 Task CRUD 엔드포인트 만들기
2. 태스크와 관련 엔티티를 영구 저장할 수 있도록 DB 연동하기
3. 백그라운드 작업 또는 메시지 큐 기반 워크플로우 도입하기
4. 자주 조회하는 태스크 목록에 캐시 적용하기
5. AI 기반 태스크 보조 기능 연결하기

## 생각 중인 AI 기능

프로젝트가 확장되면서 AI는 다음과 같은 역할을 맡을 수 있습니다.

- 모호한 태스크 설명을 더 명확한 실행 항목으로 바꾸기
- 큰 작업 하나를 더 작은 하위 작업으로 분해하기
- 지연되었거나 완료된 작업을 요약하기
- 태스크 설명을 바탕으로 우선순위를 제안하기

## 기술 방향

구현 세부사항은 진행하면서 바뀔 수 있지만, 현재 생각하는 방향은 다음과 같습니다.

- FastAPI 기반 백엔드 API
- 영속성을 위한 관계형 데이터베이스
- 비동기 처리를 위한 메시지 큐
- 자주 조회되는 데이터를 위한 캐시 계층
- 지능형 기능을 위한 AI 모델 연동 레이어

## 현재 상태

이 저장소는 이제 막 시작하는 단계입니다.

코드베이스는 한 번에 완성하는 대신, 각 단계를 직접 구현하고 이해하는 방식으로 점진적으로 만들어 갈 예정입니다.

## 개발 철학

Flow List는 의도적으로 손으로 직접 만드는 프로젝트입니다.

목적은 단순히 앱을 완성하는 것이 아니라, 백엔드 시스템이 어떻게 구성되는지, 책임이 어떻게 분리되는지, 그리고 현대적인 애플리케이션 구성 요소들이 실제로 어떻게 함께 동작하는지를 이해하는 데 있습니다.
